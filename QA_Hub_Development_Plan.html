5. AI Test Assistant

โมดูล AI ทำหน้าที่ช่วยวิเคราะห์ข้อมูลและเตรียม Draft ให้ทีม QA ไม่ใช่ตัดสินใจหรือเผยแพร่ Test Case อัตโนมัติโดยไม่มีผู้ตรวจสอบ

Requirement Analyzer

รับ Requirement, Acceptance Criteria, Product, Module และเงื่อนไขธุรกิจ แล้วสร้าง Test Scenario และ Test Case แบบ Positive, Negative, Boundary, Permission, Integration และ Regression

Error Analyzer

รับ Error Message, Stack Trace, SQL Error, API Response, Log หรือ Screenshot แล้วสรุป Error Signature, Possible Root Cause, Recommended Checks, Bug Draft และ Test Case สำหรับ Reproduce/Regression

Markdown Documentation Analyzer

อ่านไฟล์ .md ของแต่ละ Product/Module จากการ Upload หรือ Git Repository แล้วแยก Objective, Flow, Business Rule, Validation, Permission, API, Database, Error Handling และ Known Limitation เพื่อสร้าง Test Scenario และ Test Case พร้อม Source Reference

Review & Governance

ทุกผลลัพธ์ต้องอยู่ในสถานะ AI Draft แสดง Confidence Score, Assumption, Missing Information และผ่าน QA Review ก่อนบันทึกเป็น Test Case หรือ Bugจริง

Workflow: Requirement → Test Case

เลือก Product / Module
→ เลือกหรือวาง Requirement
→ AI แยก Business Rule และ Acceptance Criteria
→ Generate Scenario และ Test Case
→ QA ตรวจแก้ / เลือกเฉพาะรายการ
→ QA Lead Review
→ Approved และ Publish เข้าคลัง Test Case

Workflow: Markdown Documentation → Test Case

Upload .md หรือเชื่อม Git Repository
→ เลือก Product / Module / Branch / Version
→ Parse Heading, Table, Code Block และ Metadata
→ AI แยก Objective, Flow, Business Rule, Validation และ Dependency
→ แสดงข้อมูลที่ขัดแย้งหรือยังไม่ครบ
→ สร้าง Scenario, Test Case, Test Data และ Regression Scope
→ QA ตรวจสอบ Source Reference และ Assumption
→ Approve และ Publish เข้าคลัง Test Case

Workflow: Document Change Impact

Dev Commit การแก้ไขไฟล์ .md
→ ระบบ Sync Commit ล่าสุดและคำนวณ Diff
→ AI วิเคราะห์ Business Rule ที่เพิ่ม แก้ไข หรือลบ
→ Mapping กับ Test Case เดิม
→ แจ้ง Test Case ที่ได้รับผลกระทบ
→ เสนอ Test Case ใหม่ / Test Case ที่ต้องแก้ / Regression Scope
→ QA Review ก่อน Update Version

Workflow: Error → Bug และ Regression Test

วาง Error / Upload Log / API Response
→ Mask Password, Token และข้อมูลส่วนบุคคล
→ AI วิเคราะห์ Error Signature และสาเหตุที่เป็นไปได้
→ สร้าง Bug Draft + Steps to Reproduce
→ สร้าง Reproduction Test และ Regression Test
→ QA ตรวจสอบกับ Build / Environment จริง
→ บันทึก Bug และ Test Case

ข้อมูลผลลัพธ์ที่ AI ต้องสร้าง

หมวดข้อมูล
Test DesignScenario, Title, Preconditions, Test Data, Steps, Expected Result, Type, Priority
AnalysisBusiness Rules, Risk, Coverage Warning, Assumption, Missing Information, Confidence Score
Error AnalysisError Signature, Component, Possible Root Cause, Recommended Checks, Reproduce Steps
TraceabilityProduct, Module, Requirement ID, Build, Environment, Source Error, Generated By
Document SourceRepository, Folder, File Path, Branch, Commit Hash, Document Version, Section Heading, Line Range
Change ImpactAdded Rule, Modified Rule, Removed Rule, Impacted Test Cases, Suggested Regression Scope

สถานะ AI Draft

Generated → Pending Review → Needs Revision → Approved → Published
                         └→ Rejected

Checklist การพัฒนา AI Test Assistant

Input

  • เลือก Product และ Module
  • เลือก Requirement จากระบบ
  • วางข้อความ Requirement
  • Upload Markdown
  • เลือก Folder จาก Repository
  • เลือก Branch / Commit
  • Upload Log
  • วาง SQL/API Error
  • รองรับ Screenshot

Generation

  • สร้าง Positive Case
  • สร้าง Negative Case
  • สร้าง Boundary Case
  • สร้าง Permission Case
  • สร้าง Integration Case
  • สร้าง Regression Case

Review

  • แก้ไข Draft ได้
  • เลือกบาง Test Case ได้
  • Reject พร้อมเหตุผล
  • แสดง Confidence
  • แสดง Assumption
  • Diff ก่อน Publish

Documentation & Git

  • รองรับ .md หลายไฟล์
  • กำหนด Folder ต่อ Product
  • Sync Repository แบบ Manual/Webhook
  • อ่าน Commit Hash
  • เปรียบเทียบ Version Diff
  • สร้าง Source Citation ระดับ Section/Line
  • Mapping Document กับ Test Case
  • แจ้ง Conflict และ Missing Information

Security

  • Mask Password/Token
  • Mask PII
  • กำหนด Data Retention
  • Prompt Audit Log
  • จำกัด Product Access
  • เก็บ Repository Token ใน Secret Store
  • ห้าม Execute SQL อัตโนมัติ

Definition of Done

  • AI สร้าง Test Case Draft จาก Requirement ได้
  • AI สร้าง Bug Draft และ Regression Test จาก Error ได้
  • AI อ่านไฟล์ Markdown และสร้าง Test Case Draft พร้อม Source Reference ได้
  • ระบบ Sync Repository และระบุ Branch / Commit Hash ของเอกสารได้
  • ระบบวิเคราะห์ Diff และแจ้ง Test Case ที่ได้รับผลกระทบได้
  • ผลลัพธ์ทุกชิ้น Trace กลับไปยัง Source ได้
  • QA แก้ไข เลือก Reject และ Approve ได้
  • ระบบ Mask Secret ก่อนส่งให้ AI
  • บันทึก Prompt, Model, ผู้ใช้งาน และผลการ Review ใน Audit Log
  • AI ไม่สามารถ Publish หรือ Execute คำสั่งอันตรายโดยไม่มี Human Approval
